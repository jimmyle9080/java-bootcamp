The student management console app is under the package com.academy.student within the folder path src/com/academy/student matching package name so it compiles. I used one shared scanner on system.in, injected into StudentManager, and read everything with nextLine() + Integer.parseINt/Double.parseDouble to avoid leftover bug that happens when you mix nextInt() with nextLine(). Students are stored in a fixed-size Student[] array with a separate studentCount, and every loop runs only to studentCount (not students.length) so it never touches empty null slots. Main stays thin  it just shows the menu and delegates  while StudentManager owns add, display, search, and average. Validation rejects bad IDs, duplicates, empty names, and out-of-range marks before anything is stored.
